# System Design Interview Guide

Welcome to the System Design Interview Guide. This repository is designed to take you from basic concepts to advanced High-Level Design (HLD) case studies, specifically tailored for technical interviews.

## Structure

The repository is organized into the following sections:

### [01. Fundamentals](./01-fundamentals/)

Core concepts that form the building blocks of any distributed system.

- [01. Networking Basics](./01-fundamentals/01-networking-basics.md) (TCP/UDP, HTTP Protocols)
- [02. Scalability Basics](./01-fundamentals/02-scalability-basics.md) (Vertical vs Horizontal, Latency vs Throughput)
- [03. CAP Theorem](./01-fundamentals/03-cap-theorem.md)
- [04. Database Scaling](./01-fundamentals/04-database-scaling.md) (Sharding, Replication, Consistent Hashing)
- [05. API Design](./01-fundamentals/05-api-design.md) (REST, GraphQL, gRPC)
- [06. Microservices Architecture](./01-fundamentals/06-microservices.md)
- [07. Reliability Patterns](./01-fundamentals/07-reliability-patterns.md)
- [08. Observability](./01-fundamentals/08-observability.md)
- [09. Security](./01-fundamentals/09-security.md) (Encryption, Hashing, OWASP, CORS)
- [10. Authentication](./01-fundamentals/10-authentication.md)
- [11. Real-Time Systems](./01-fundamentals/11-real-time-systems.md)

### [02. Components](./02-components/)

Deep dives into specific infrastructure components.

- [01. Load Balancers](./02-components/01-load-balancers.md)
- [02. Databases (SQL vs NoSQL)](./02-components/02-databases-sql-vs-nosql.md)
- [03. Caching Strategies](./02-components/03-caching.md)
- [04. Message Queues](./02-components/04-message-queues.md) (Kafka vs RabbitMQ)
- [05. Rate Limiting](./02-components/05-rate-limiting.md)

### [03. High-Level Design (HLD) Case Studies](./03-hld-case-studies/)

Real-world system design problems often asked in interviews.

- [01. Scaling from Zero to Millions](./03-hld-case-studies/01-scaling-from-zero-to-millions.md)
- [02. Design a URL Shortener (TinyURL)](./03-hld-case-studies/02-design-url-shortener.md)

### [04. Interview Guide](./04-interview-guide/)

A framework for approaching system design questions during the actual interview.

- [01. The Framework](./04-interview-guide/01-framework.md)

## How to use this repo

Start with the **Interview Guide** to understand the structure of a system design interview. Then, review **Fundamentals** and **Components** to ensure you have the necessary tools. Finally, practice with the **HLD Case Studies**.
